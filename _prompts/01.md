Build a minimal, BUILDABLE-AND-RUNNABLE Maven project that reproduces a legacy
enterprise Java stack, so I can later run modernization/migration recipes against it.

## Goal
- A "hello world" Spring Boot app: ONE @SpringBootApplication class + ONE @RestController
  with a GET "/" returning "hello world", plus one trivial JUnit 4 test.
- It must compile, `mvn clean package` must succeed, and the app must start.
- `mvn dependency:tree` should produce a stack SIMILAR to the target below (same major
  frameworks/libraries and versions). It does NOT need to be byte-identical — same stack, not same tree.

## Hard constraints
- Java 8 (source/target 1.8).
- Packaging: jar. groupId com.acme, artifactId my-product, version 1.0.0.
- Use BOMs/parents where possible, explicit versions where the stack forces overrides.
